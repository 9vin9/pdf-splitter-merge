# ✂ PDFTools

브라우저에서 바로 사용하는 PDF 분할 · 병합 도구입니다.  
서버 없이 로컬에서 동작하며, 파일이 외부로 전송되지 않습니다.

---

## 기능

### ✂ PDF 분할
- PDF 파일 업로드 후 페이지 미리보기
- 썸네일 호버 → **시작 / 끝 버튼**으로 범위 선택
- 선택 완료 시 범위 자동 추가
- 숫자 직접 입력으로도 범위 지정 가능
- 여러 범위를 한 번에 등록 후 일괄 분할
- 다운로드 전 파일명 인라인 수정
- 분할 기록 저장 및 설정 복구

### ⊕ PDF 병합
- 여러 PDF 파일 동시 업로드
- 드래그 앤 드롭으로 순서 변경
- 썸네일 클릭으로 첫 페이지 미리보기
- 병합 결과 파일명 기본값: `파일A + 파일B + 파일C`
- 다운로드 전 파일명 수정 가능

---

## 시작하기

### 로컬 서버 실행 (필수)

`file://` 프로토콜로 직접 열면 PDF.js 워커 보안 에러가 발생합니다.  
반드시 로컬 서버를 통해 실행하세요.

```bash
# Node.js
npx serve .

# Python 3
python3 -m http.server 8000
```

브라우저에서 `http://localhost:8000` 접속

## 파일 구조

```
pdf-splitter-merge/
├── index.html          # 분할 페이지
├── script.js           # 분할 로직
├── style.css           # 분할 페이지 스타일
├── merge.html          # 병합 페이지
├── merge-script.js     # 병합 로직
├── merge-style.css     # 병합 페이지 스타일
├── shared.css          # 공통 스타일 (헤더, 버튼, 토스트 등)
├── pdf.min.js          # PDF.js 라이브러리 (로컬)
└── pdf.worker.min.js   # PDF.js 워커 (로컬)
```

## 주요 개선 이력

- 탭 네비게이션으로 분할/병합 전환 (분할 파랑 · 병합 보라)
- 썸네일 호버 시작/끝 버튼으로 범위 선택 UX 개선
- 미리보기 패널 sticky 고정 (썸네일 스크롤과 분리)
- 다운로드 전 파일명 인라인 수정 기능
- 병합 기본 파일명 `+` 연결 자동 생성
- `alert()` 전체 제거 → 토스트 알림 교체
- 한글 PDF 렌더링 오류 수정 (`cMapUrl` 설정)
- `shared.css` 분리로 두 페이지 디자인 일관성 확보
